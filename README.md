# 專案管理期末考試系統

一個功能完整的線上考試系統，專為專案管理課程設計，提供智能化的考試體驗與詳細的學習分析。

## ✨ 核心特色

### 🎯 智能考試系統
- **多題庫支援**：自動載入多個題目文件，智能備援機制
- **格式自動修復**：自動識別並修正題目格式問題
- **漸進式題目載入**：優先載入完整版題目，確保最佳考試體驗
- **錯誤容忍機制**：當主要題目文件無法載入時自動切換備用文件

### 🚀 先進的考試功能
- **靈活配置選項**：
  - 隨機題目順序：每次考試都有不同的題目排列
  - 答案解釋顯示：可選擇是否在結果中顯示詳細解釋
- **實時計時追蹤**：精確記錄考試時間，分析答題速度
- **智能進度保存**：
  - 自動保存功能（每30秒）
  - 意外關閉保護
  - 24小時內進度恢復
- **無障礙設計**：完整的鍵盤快捷鍵支援

### 📊 深度學習分析
- **多維度統計**：分數、正確率、用時、平均速度
- **個性化錯誤分析**：
  - 識別答錯的題目類型
  - 提供針對性複習建議
  - 錯題重點標示
- **時間效率分析**：
  - 平均每題用時計算
  - 答題速度優化建議
  - 時間管理指導
- **歷史記錄追蹤**：保存最近10次考試記錄，追蹤學習進度

### 🎨 現代化用戶介面
- **響應式設計**：完美適配桌面、平板和手機
- **深色模式支援**：自動適配系統主題偏好
- **直觀的進度顯示**：視覺化進度條和題目導航
- **流暢的動畫效果**：提升用戶體驗的微互動

- **complete_exam_questions.json**: 50題專案管理題目，包含完整解釋
- **full_exam_questions_corrected.json**: 備用題庫，確保系統穩定性

## 🎮 使用指南

### 快速開始
1. **開啟應用程式**
   - 直接在瀏覽器中開啟 `index.html`
   - 系統會自動載入題目並驗證格式

2. **配置考試設定**
   - **隨機題目順序**：每次考試都有不同的題目排列
   - **顯示答案解釋**：在結果頁面中顯示詳細解釋

3. **開始考試**
   - 點擊「開始考試」按鈕
   - 系統會自動開始計時並載入第一題

### 考試過程
- **答題方式**：
  - 點擊選項按鈕
  - 使用數字鍵 1-4 快速選擇
- **題目導航**：
  - 使用「上一題」/「下一題」按鈕
  - 方向鍵 ← → 快速導航
- **進度追蹤**：
  - 即時顯示答題進度
  - 自動保存答題狀態
- **時間管理**：
  - 實時顯示考試時間
  - 無時間限制，專注於學習

### 結果分析
考試結束後，系統提供全面的分析報告：

#### 📈 成績統計
- **總分與及格狀態**：即時計算分數（60分及格）
- **正確率分析**：詳細的答題準確度
- **時間效率**：總用時和平均每題用時

#### 🔍 錯誤分析
- **錯題清單**：列出所有答錯的題目
- **詳細解釋**：每題的正確答案和解釋說明
- **複習建議**：針對性的學習建議

#### ⏱️ 時間分析
- **答題速度評估**：
  - 快速（<30秒/題）：建議更仔細思考
  - 適中（30-120秒/題）：理想的答題速度
  - 較慢（>120秒/題）：可以提高效率

#### 📚 歷史記錄
- 保存最近10次考試記錄
- 追蹤學習進度和改善情況
- 支援歷史成績對比分析

## 📊 題目格式規範

本系統支援標準化的JSON題目格式，確保題目的一致性和可讀性：

```json
{
  "id": 1,
  "question": "某專案經理在評估專案對環境的影響時，主動尋找減少碳足跡的替代材料，並確保所有利害關係人的利益都被公平考量。這種行為符合哪一項專案管理原則？",
  "options": [
    "A. 協作",
    "B. 管家式服務", 
    "C. 價值",
    "D. 品質"
  ],
  "answer": "B",
  "explanation": "管家式服務原則強調考量資源與利害關係人利益，專案經理主動尋找環保材料並公平考量所有利益方，正是此原則的體現。"
}
```

### 欄位說明
- **id**: 題目唯一識別碼（數字）
- **question**: 題目內容（字串）
- **options**: 選項陣列，包含A-D四個選項
- **answer**: 正確答案（A、B、C或D）
- **explanation**: 答案解釋（可選，提供學習指導）

### 自動修復機制
系統會自動處理以下格式問題：
- 缺少題目ID：自動分配序號
- 選項格式不一致：統一為「字母. 內容」格式
- 缺少必要欄位：使用預設值填充
- 無效的答案選項：預設為選項A

## 🐛 疑難排解

### 常見問題解決

#### 題目載入失敗
**現象**：頁面顯示「載入中...」或錯誤訊息

**解決方案**：
1. **檢查文件完整性**
   ```bash
   # 確認以下文件存在於同一目錄
   index.html
   app.js
   style.css
   complete_exam_questions.json
   full_exam_questions_corrected.json
   ```

2. **驗證JSON格式**
   - 使用線上JSON驗證工具檢查題目文件格式
   - 確認文件編碼為UTF-8

3. **瀏覽器相容性**
   - 使用現代瀏覽器（Chrome 70+、Firefox 65+、Safari 12+、Edge 79+）
   - 避免使用Internet Explorer

#### 進度保存失敗
**現象**：考試進度無法恢復，或顯示保存失敗訊息

**解決方案**：
1. **檢查瀏覽器設定**
   - 確認瀏覽器支援localStorage
   - 避免使用隱私模式/無痕模式
   - 檢查瀏覽器的資料保存設定

2. **清理瀏覽器資料**
   ```
   Chrome: 設定 → 隱私權和安全性 → 清除瀏覽資料
   Firefox: 設定 → 隱私權與安全性 → 清除資料
   Safari: 偏好設定 → 隱私權 → 管理網站資料
   ```

3. **儲存空間檢查**
   - 確認裝置有足夠的儲存空間
   - 清理不需要的瀏覽器資料

#### 頁面顯示異常
**現象**：樣式錯亂、按鈕無法點擊、響應式問題

**解決方案**：
1. **強制重新整理**
   - Windows: `Ctrl + F5`
   - macOS: `Cmd + Shift + R`

2. **檢查CSS載入**
   - 開啟開發者工具（F12）
   - 檢查Console是否有CSS載入錯誤
   - 確認style.css文件路徑正確

3. **瀏覽器重置**
   - 清除瀏覽器快取
   - 重新啟動瀏覽器

#### 鍵盤快捷鍵無效
**現象**：方向鍵或數字鍵無法正常工作

**解決方案**：
1. **確認焦點狀態**
   - 點擊考試頁面的任何位置
   - 確保考試頁面處於活動狀態

2. **檢查瀏覽器衝突**
   - 關閉可能衝突的瀏覽器擴充功能
   - 嘗試使用滑鼠點擊答題

2. **Clone** 到本地開發環境
3. 建立功能分支：`git checkout -b feature/amazing-feature`
4. 提交變更：`git commit -m '新增驚人功能'`
5. 推送分支：`git push origin feature/amazing-feature`
6. 建立 **Pull Request**

#### 📝 文件改善
- 修正錯字或語法錯誤
- 增加使用範例
- 改善說明的清晰度
- 翻譯成其他語言

### 開發規範

#### 程式碼風格
- 使用 2 空格縮排
- 函數和變數使用 camelCase
- 類別使用 PascalCase
- 常數使用 UPPER_SNAKE_CASE

#### 提交訊息格式
```
類型(範圍): 簡短描述

詳細說明（可選）

Footer（可選）
```

範例：
```
feat(exam): 新增考試時間限制功能

- 添加計時器設定選項
- 實作時間到自動提交
- 更新使用者介面

Closes #123
```
